Why we needed this

Legacy HR doesn’t handle mergers well. Every acquisition meant months of migration, incompatible systems, duplicate records, and HR teams drowning in manual reconciliation, with no combined workforce metrics during due diligence.

  • Months of data migration per target
  • Systems that couldn’t talk to each other
  • Duplicate employee records across platforms
  • No visibility into combined workforce metrics
  • Manual reconciliation blocking deal teams

We built for transaction readiness from day one, not bolt-on reporting after the fact.

The M&A problem we solved

Traditional approach

Acquired Co A → Their HRIS → Migrate to ours → 6 to 18 mo
Acquired Co B → Different HRIS → Another migration → +6 mo
  • Each integration is a one-off project
  • Data quality varies wildly
  • No standard data model
  • HR can’t answer basic diligence questions

Our approach

Acquired Co → API connect → Instant visibility
→ Standardized model → Dashboards → Transaction reporting
  • Reusable integration pattern
  • Validation and confidence at ingestion
  • Normalized schema across targets
  • Real-time answers for deal teams

Key product requirements

1. Universal adapter pattern

Integration layer is system-agnostic. Frappe HRMS is the reference implementation; adapters extend to Workday, SuccessFactors, ADP, and regional payroll.

2. Normalized data model

Map job titles to families, compensation to currency-neutral views, org hierarchy flexibly, and employment types to consistent categories.

3. Transaction-ready dashboards

Headcount by country, total comp burden, key performers to retain, attrition risk: instant answers for diligence, not ad-hoc reports.

4. Audit & compliance

Historical preservation, full audit trails, GDPR-aware handling, and document retention built in from the start.

5. Separation of concerns

Payroll stays with specialists. We pull, normalize, and analyze; Frappe handles payroll complexity so we focus on M&A-specific analytics.

What discovery showed

M&A is a different job-to-be-done

Focus groups with HR ops and deal-team stakeholders during live transactions surfaced a split from day-to-day HR: users needed rapid multi-source ingestion, flexible schema mapping, and diligence-ready rollups, not another ESS portal. Opportunity maps ranked transaction visibility above employee self-service for the first release.

Integration beat replacement

Workflow interviews and a build-vs-buy matrix killed the “replace every payroll system” hypothesis. Stakeholders wanted source systems to stay authoritative; the product should connect, normalize, and report. That research drove the adapter pattern and Frappe for payroll depth instead of rebuilding compliance engines.

Data trust was the workflow blocker

Shadowing reconciliation during diligence showed duplicate records and inconsistent fields were why teams reverted to spreadsheets. Tree tests on dashboard tasks (headcount by country, comp burden, retention risk) failed when numbers disagreed across sources. We prioritized validation at ingestion, duplicate handling, and confidence scores before adding net-new analytics.

Adoption had to survive deal crunch

HR leads in post-close integration said they would not learn a new tool mid-transaction unless paths were obvious under pressure. Prototype walkthroughs and tree tests on core diligence flows informed a minimal dashboard set and plain-language labels, so change management was not blocked by navigation.

PM takeaways

  • Solve real M&A pain, not features because they’re cool
  • Embrace integration; no single system wins every workflow
  • Design for change; deal context shifts weekly
  • Start with the exit; build for sale and separation from day one
  • Listen to HR: they surface what’s broken in incumbent tools
